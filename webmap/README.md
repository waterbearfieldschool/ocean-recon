# Ocean Recon node map

A static single-page site that plots node positions from the bayou feed
(https://bayou.pvos.org/data/995ywq4zg2iq) on the VerD-style USNG basemap.

## Pages

- `index.html` — **mission replay** (the shareable page): reads the local
  `data.json` snapshot, shows every node's track with a time scrubber and
  play button. Fully static — no live feed needed.
- `live.html` — live mode: polls the bayou feed directly (full-feed scan to
  discover nodes, then `latest/<node_id>` every 5 s). Needs the feed online.

Both draw USNG grid labels pinned to the viewport edges (visible at any
zoom), show a mouse-follow USNG readout, colored per-node markers with
breadcrumb trails, and a sidebar with each node's 100 m square — the same
"013-284" digits the MeshCore radios display.

## Update the data snapshot

```
python3 fetch_data.py        # rewrites data.json from the bayou feed
```

## Serve locally

```
cd webmap
python3 -m http.server 8000  # then open http://localhost:8000/
```

## Publish (GitHub Pages)

The folder is self-contained apart from the Leaflet/proj4 CDN scripts, so any
static host works. For GitHub Pages the simplest routes are:

- Settings → Pages → "Deploy from a branch" with the `/docs` folder, after
  `git mv webmap docs` (or copying this folder to `docs/`), or
- a `gh-pages` branch / Actions workflow that publishes `webmap/`.

`index.html` (the replay) is what visitors land on; `live.html` also works
from Pages since the bayou feed sends open CORS headers.

## How the basemap works

- `usng-base*.png` + `.json` are generated by `gridmap/make_map.py --web`:
  the PNG is rendered edge-to-edge so its pixels map linearly onto the UTM
  extent recorded in the JSON sidecar. The `-plain` variant
  (`--no-edge-labels`) has no baked-in grid labels because the pages draw
  their own.
- The pages show the image in Leaflet's flat `CRS.Simple` mode and use
  proj4js to convert `gps_lat`/`gps_lon` to UTM zone 19N, so marker placement
  is exact (no web-mercator reprojection of the basemap).

Regenerate:

```
.venv/bin/python gridmap/make_map.py \
    --bounds 41.765 -71.397 41.794 -71.3655 \
    --bw --master-only --no-howto --no-footer \
    --provider OpenStreetMap.HOT --posterize 4 --invert-water \
    --web --no-edge-labels --out webmap --name usng-base-plain
```

Any `make_map.py` style flags work (drop `--invert-water` for gray water,
add `--latlon` for a graticule, change `--bounds`, etc.). If you change
`--bounds`, the pages pick up the new extent from the JSON sidecar
automatically.
