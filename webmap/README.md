# Ocean Recon live node map

A single-page site that plots live node positions from the bayou feed
(https://bayou.pvos.org/data/995ywq4zg2iq) on the VerD-style USNG basemap.

## Serve it

```
cd webmap
python3 -m http.server 8000
```

then open http://localhost:8000/ . Internet access is needed at runtime for
the bayou feed and the Leaflet/proj4 CDN scripts (the basemap itself is local).

## How it works

- `usng-base.png` + `usng-base.json` are generated by `gridmap/make_map.py --web`:
  the PNG is rendered edge-to-edge so its pixels map linearly onto the UTM
  extent recorded in the JSON sidecar.
- `index.html` shows that image in Leaflet's flat `CRS.Simple` mode and uses
  proj4js to convert each node's `gps_lat`/`gps_lon` to UTM zone 19N — so
  marker placement is exact (no web-mercator reprojection of the basemap).
- Nodes are discovered from the full feed (`/json/`, rescanned every 2 min)
  and then their `/latest/<node_id>` endpoints are polled every 5 s.
- Each node gets a colored marker, ID label, dashed breadcrumb trail, and a
  sidebar card with its 100 m USNG square (the same "013-284" digits the
  MeshCore radios show). Nodes with fixes outside the map are listed as
  OFF MAP. Clicking the map shows the USNG coordinate of the click.

## Regenerate the basemap

```
.venv/bin/python gridmap/make_map.py \
    --bounds 41.765 -71.397 41.794 -71.3655 \
    --bw --master-only --no-howto --no-footer \
    --provider OpenStreetMap.HOT --posterize 4 --invert-water \
    --web --out webmap --name usng-base
```

Any `make_map.py` style flags work here (drop `--invert-water` for gray water,
add `--latlon` for the graticule, change `--bounds`, etc.).
